# <a name="management-agent-run-error-codes"></a>Kody błędów uruchamiania agenta zarządzania
W poniższych tabelach przedstawiono kody błędów dla Synchronization Service Manager interfejsu użytkownika w Microsoft Identity Manager (MIM) 2016. Szczegółowe opisy zostały podane w celu wyjaśnienia każdego błędu.

## <a name="connection-errors"></a>Błędy połączenia

|Błąd | Opis |
|----|----|  
| nie można nawiązać połączenia&#8209;         | Połączenie z połączonym katalogiem nie powiodło się z przyczyn innych niż uwierzytelnianie. Na przykład sieć jest niedostępna lub serwer docelowy jest w trybie offline. |
| porzucono połączenie&#8209;        | Połączenie między agentem zarządzania i połączonym katalogiem już nie istnieje. W wielu przypadkach agent zarządzania próbuje ponownie nawiązać połączenie z podłączonym katalogiem.  |
| niepowodzenie uwierzytelniania&#8209;     | Uwierzytelnianie nie jest możliwe przy użyciu podanych poświadczeń.  |
| Niepowodzenie&#8209;uprawnienia         | Brak wystarczających praw dostępu do kontenera w połączonym katalogu. Ten błąd jest oczekiwany tylko w przypadku agentów zarządzania LDAP (Lightweight Directory Access Protocol), które przeszukują różne kontenery podłączonych katalogów.  |
| nie można przeszukać&#8209;            | Wyszukiwanie kontenera lub tabeli nie powiodło się z powodu nieoczekiwanego błędu. |
| Ostrzeżenie&#8209;nie&#8209;znaku wodnego      | Agent zarządzania nie może odczytać znaku wodnego podczas wykonywania pełnego importu. Ten błąd jest oczekiwany tylko w przypadku agenta zarządzania dla serwera Sun z katalogiem 5,1 (dawniej iPlanet Directory Server), gdy początkowa konfiguracja agenta zarządzania zostanie ukończona, a połączony katalog ma włączony dziennik zmian. Później, gdy dziennik zmian podłączonych katalogów jest wyłączony, jeśli konfiguracja agenta zarządzania nie zostanie zaktualizowana, ostrzeżenie zostanie przeprowadzone po zakończeniu pełnego importu. |
| nie&#8209;uruchomić&#8209;partycji&#8209;Usuń | Partycja LDAP, która została pierwotnie skonfigurowana dla tego agenta zarządzania, już nie istnieje. Ten błąd jest zwracany, gdy partycja została usunięta, lub partycja została usunięta i ponownie utworzona o tej samej nazwie. W tym drugim przypadku, nawet jeśli nazwa jest taka sama, identyfikator GUID partycji błędów został zmieniony. |

## <a name="discovery-errors"></a>Błędy odnajdywania

|Błąd | Opis |
|----|----|  
| Brak typu&#8209;zmiany&#8209;      | Ten błąd jest zwracany podczas importowania różnicowego przez agentów zarządzania opartego na plikach i bazami danych, a także agenta zarządzania dla serwerów z systemem Sun i Netscape, gdy wartość kolumny Typ zmiany (Dodaj, Modyfikuj, Usuń) nie istnieje. |
| Nieprawidłowy typ&#8209;zmiany&#8209;      | Ten błąd jest zwracany podczas importowania różnicowego przez agentów zarządzania opartego na plikach i bazami danych, a także agenta zarządzania dla serwerów Sun i Netscape Directory, gdy wartość kolumny Typ zmiany nie jest zgodna z listą prawidłowych typów zmian. Jest również zwracany z pełnego importu protokołu LDAP Data Interchange Format (LDIF), gdy pole typu zmiany jest obecne i ma wartość inną niż Dodaj. |
| &#8209;zmiany&#8209;wartości&#8209; | Ten błąd jest zwracany podczas importu różnicowego przez agentów zarządzania serwerów plików i Sun oraz Netscape, gdy występuje więcej niż jedna wartość dla typu zmiany. |
| wymagany&#8209;pełny&#8209;obiektu        | Ten błąd jest zwracany podczas wykonywania importu różnicowego agenta zarządzania opartego na plikach lub podczas wznawiania z poziomu agenta zarządzania opartego na plikach. Wskazuje on, że agent zarządzania przesłał modyfikację obiektu, który nie może znajdować się w obszarze łącznika. Aparat synchronizacji żąda bieżących wartości wszystkich atrybutów obiektu. Ponieważ jest to import z pliku, te informacje są niedostępne. Pełny import powinien rozwiązać ten problem. |
| Brak&#8209;DN              | Ten błąd jest zwracany w przypadku agentów zarządzania opartych na plikach (czyli agentów zarządzania dla plików LDIF, DSML lub Flat ze skonfigurowanymi atrybutami nazw domen), jeśli nie ma wartości nazwy domeny. Ta wartość jest również zwracana dla uszkodzonego dziennika zmian serwera katalogowego Sun, gdzie brakuje atrybutu nazwy domeny. Wskazuje on, że agent zarządzania może odczytać element i przeanalizować go, ale nie ma żadnej wartości nazwy domeny dla tego obiektu. |
| nazwa DN&#8209;nie&#8209;LDAP&#8209;zgodne   | Ten błąd jest zwracany, gdy agent zarządzania dla protokołu LDAP, LDIF, DSML lub plik prosty ze skonfigurowanym atrybutem nazwy domeny zgłasza wartość nazwy domeny, która nie jest zgodna ze specyfikacją LDAP. |
| Nieprawidłowa nazwa DN&#8209;              | Ten błąd jest zwracany, gdy agent zarządzania zgłasza, że nazwa domeny nie spełnia ograniczenia programu FIM, co obejmuje:   <br/><ul><li>Co najmniej jeden znak, który nie jest dozwolony przez program FIM.</li><li>Pusta względna nazwa wyróżniająca (znana również jako RDN). </li><li>Względna nazwa wyróżniająca, która przekracza wartość maksymalną dla programu FIM. </li><li>Liczba poziomów hierarchii nazwy domeny przekracza wartość maksymalną dla programu FIM.</li></ul> |
| Brak&#8209;kotwicy&#8209; | Ten błąd jest zwracany przez agentów zarządzania plikami i bazami danych, a także agenta zarządzania dla serwerów z systemem Sun i Netscape, gdy nie można utworzyć kotwicy, ponieważ co najmniej jeden atrybut reguły konstrukcji kotwicy nie ma wartości. |
| składnik&#8209;zakotwiczenie&#8209;&#8209; | Ten błąd jest zwracany przez agenta zarządzania dla serwerów Sun i Netscape Directory, jeśli nie można skonstruować kotwicy, ponieważ atrybut reguły konstrukcji kotwicy ma więcej niż jedną wartość. |   
|zakotwiczenie&#8209;zbyt&#8209;długa | Ten błąd jest zwracany przez agentów zarządzania plikami i bazami danych, a także agenta zarządzania dla serwerów z systemem Sun i Netscape, gdy konstrukcja kotwicy tworzy kotwicę, która przekracza maksymalny limit rozmiaru programu FIM. |
|zduplikowany obiekt&#8209; | Ten błąd jest zwracany w przypadku pełnego importu przez agentów zarządzania opartymi na plikach i bazami danych, gdy obiekt z tą samą kotwicą został już zgłoszony do aparatu synchronizacji podczas tego przebiegu.<br/>**Uwaga** : niezgodność obiektów przestrzeni łącznika występuje tylko wtedy, gdy bieżący krok przebiegu zakończył się pomyślnie, Zakończ z błędem synchronizacji, Ukończ z ostrzeżeniem lub Zakończ z chwilą. |
|Brak klasy&#8209;&#8209;obiektu | Ten błąd jest zwracany przez agenta zarządzania oparty na plikach (czyli agenta zarządzania dla DSML, LDIF lub pliku prostego ze skonfigurowanym atrybutem klasy obiektów) lub agenta zarządzania dla serwerów Sun i Netscape Directory, jeśli istnieje uszkodzony dziennik zmian. Oznacza to, że agent zarządzania nie może odczytać wartości atrybutu klasy obiektu. |
|Brak typu&#8209;obiektu&#8209; | Ten błąd jest zwracany podczas wznawiania importowania z uszkodzonego pliku upuszczania. Ten błąd nie został napotkany podczas normalnego działania.        |
|Napotkano-typ obiektu  | Ten błąd jest zwracany przez agenta zarządzania opartego na plikach, gdy odczytuje obiekt, który ma zestaw wartości klasy obiektów, których nie można dopasować do żadnego z mapowań prefiksu. |
|Błąd analizy&#8209; | Ten błąd jest zwracany przez agenta zarządzania dla serwerów katalogów Sun i Netscape w trybie różnicowym oraz za pomocą agentów zarządzania opartych na plikach, gdy nie mogą przeanalizować wpisu. `<entry-number>`Element (i w większości przypadków `<line-number>` `<column-number>` ) jest obecny, aby ułatwić znalezienie błędu. `<attribute-name>`Element może być obecny. Po napotkaniu tego działania agent zarządzania dla serwerów usługi Sun i Netscape Directory kończy działanie. Agenci zarządzania opartymi na plikach rejestrują błąd odnajdywania i kontynuują. |   
|Błąd odczytu&#8209; | Ten błąd jest zwracany przez agentów zarządzania opartych na wywołaniach, gdy występuje błąd ogólny podczas odczytywania określonego obiektu. Zwykle powoduje to zakończenie przebiegu. Istnieje element błędu połączonego źródła danych, którego można użyć do rozwiązania problemu.    |
|błąd&#8209;przemieszczania | Ten błąd jest zwracany przez większości agentów zarządzania. Oznacza to, że aparat synchronizacji nie może przemieścić delty w przestrzeni łącznika. Serwer tworzy dziennik zdarzeń, który zawiera informacje o problemie i którego można użyć do rozwiązywania problemów. Większość agentów zarządzania kontynuuje importowanie, gdy błąd jest rejestrowany, ale agent zarządzania dla Sun i Netscape przebiegów Delta jest zatrzymywany. Luki w przetwarzaniu dziennika zmian mogą spowodować niespójny stan w obszarze łącznika. Ten błąd nie został napotkany podczas normalnego działania.  |
|Nieprawidłowy typ&#8209;modyfikacji&#8209; | Ten błąd jest zwracany podczas importu różnicowego na agencie zarządzania LDIF, gdy typ modyfikacji poziomu obiektu nie jest jednym ze standardowych typów modyfikacji LDIF lub w obiekcie objectClass istnieje `nonreplace` Typ modyfikacji, taki jak `add: objectclass` lub `delete: objectclass` .    |
|&#8209;typy modyfikacji&#8209;powodujących konflikty        | Ten błąd jest zwracany przez agenta zarządzania LDIF wskazujący, że w tym samym rekordzie zostały napotkane różne typy modyfikacji poziomu atrybutu (w tym przypadku nazwa atrybutu, który utworzył typy powodujące konflikty) lub wiele różnic LDIF zastąpić w tym samym pliku, na przykład:    <br/> [Przykład pliku LDIF](./media/maerrorcodes/conflict-modification-types.jpg) |
|niezgodność pojedynczego&#8209;&#8209; | Ten błąd jest zwracany przez agenta zarządzania opartego na plikach, gdy raport zawiera więcej niż jedną wartość, lub więcej niż jedną wartość usuwania dla atrybutu, który jest zdefiniowany w programie FIM jako atrybut pojedynczej wartości. Ten błąd może wskazywać, że schemat połączonego źródła danych przechowywany przy użyciu programu FIM jest niepoprawnie określony (agenci zarządzania opartymi na plikach) lub nieaktualny z bieżącym schematem. Zawiera `<attribute-name>` element, aby nadać kontekstowi błędu.    |
|Nieprawidłowa wartość&#8209;atrybutu&#8209; | Ten błąd jest zwracany przez agenta zarządzania opartego na wywołaniach, gdy wartość atrybutu jest odczytywana, która nie jest zgodna z typem atrybutu zadeklarowanym w schemacie. Zawiera `<attribute-name>` element, aby nadać kontekstowi błędu.  |
|Nieprawidłowa wartość&#8209;Base64&#8209;| Ten błąd jest zwracany przez agentów zarządzania dla serwerów usługi LDIF, DSML, Sun i Netscape w przypadku wystąpienia nieprawidłowego ciągu Base64.     |
|Nieprawidłowa&#8209;wartość&#8209;liczbowej | Ten błąd jest zwracany przez agentów zarządzania opartych na plikach i agenta zarządzania dla protokołu LDAP, gdy nie można przeanalizować wartości liczbowej. Zawiera `<attribute-name>` element, aby nadać kontekstowi błędu. |
|Nieprawidłowa&#8209;wartość&#8209;logicznej | Ten błąd jest zwracany przez agentów zarządzania opartych na plikach i agenta zarządzania dla protokołu LDAP, gdy nie można przeanalizować wartości logicznej. Zawiera `<attribute-name>` element, aby nadać kontekstowi błędu.  |
|odwołanie&#8209;&#8209;nie&#8209;&#8209;zgodnych z protokołem LDAP | Ten błąd jest zwracany przez agentów zarządzania dla plików LDAP, LDIF i DSML lub zwykłych (z skonfigurowanym atrybutem nazwy domeny), jeśli wartość nazwy domeny nie jest zgodna ze specyfikacją LDAP. Ten komunikat o błędzie zawiera `<attribute-name>` element umożliwiający podanie kontekstu błędu.|
|Nieprawidłowa wartość&#8209;odwołania&#8209; | Ten błąd jest zwracany przez agenta zarządzania, jeśli nazwa domeny nie spełnia ograniczeń programu FIM, takich jak:  <br/><ul><li>Jeden lub więcej znaków, które nie są dozwolone przez program FIM</li><li>Pusta względna nazwa wyróżniająca (znana również jako RDN)  </li><li>Względna nazwa wyróżniająca, która przekracza wartość maksymalną dla programu FIM  </li><li>Liczba poziomów hierarchii nazwy domeny przekroczyła wartość maksymalną dla programu FIM.</li></ul>  |
|nieobsługiwany typ&#8209;wartości&#8209; | Ten błąd jest zwracany przez agenta zarządzania DSML lub LDIF, gdy typ wartości podanych w pliku jest niezgodny z typem atrybutu, w tym:    <br/><ul><li>Wartość identyfikatora URI lub adresu URL jest podawana dla atrybutu niebędącego ciągiem lub dla dowolnego zastrzeżonego słowa kluczowego, takiego jak nazwa DN, objectClass lub ChangeType.</li><li>Wartość Base64 jest podawana dla atrybutu ChangeType.</li><li>Wartość ciągu zawierająca znaki nienależące do zestawu znaków ASCII jest podawana dla atrybutu binarnego.</li></ul> |

| nieoczekiwany&#8209;aprowizacji&#8209;atrybutu           | Ten błąd jest zwracany przez agenta zarządzania programu Lotus Notes podczas eksportowania nowo zainicjowanego obiektu i pewnych atrybutów do aprowizacji, ustawiane przez rozszerzenie klienta, nie należy uwzględniać, ponieważ są niezgodne z wartościami innych atrybutów aprowizacji.
